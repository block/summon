# Technical Noir Dogfood Prompts

Use these prompts to compare Technical Noir generations across fingerprint versions. Each prompt should produce one generation per fingerprint version before review.

## tn-landing-proof

**Prompt:** Create a landing surface for a self-hosted agent runtime that moves tasks from local terminal work to cloud workers. Include one primary setup action, one command snippet, and proof of an agent handoff.

**Eval axis:** generation

**Claims under test:** quiet technical hierarchy, source-proof-follows-claim, command-action pair, terminal evidence, no chromatic brand accent.

**Strong output traits:** warm near-charcoal canvas; off-white primary CTA; plausible command; evidence panel with handoff state; no gradients, neon, source names, or generic SaaS card grid.

**Likely failures:** light-page recreation; fake code decoration; heavy billboard typography; colorful action system.

## tn-workspace-overview

**Prompt:** Create an agent workspace overview for three parallel code tasks: a migration, a failing test investigation, and a dependency upgrade. Show current state, evidence, and the next action.

**Eval axis:** generation

**Claims under test:** workspace shell, terminal evidence, next action visibility, readable technical density.

**Strong output traits:** task/session rail; active task; log or plan evidence; one calm next action; compact warm-dark panels with hairlines.

**Likely failures:** dashboard card pile; too many equal panels; no real technical material; colorful status dashboard.

## tn-operational-comparison

**Prompt:** Compare three ways to run an AI code migration: local laptop, cloud worker, and managed team runner. Recommend one path and explain the tradeoff using shared operational criteria.

**Eval axis:** generation

**Claims under test:** shared criteria comparison, off-white recommendation, mono metadata, no colorful scoring.

**Strong output traits:** aligned rows or compact selector plus evidence frame; criteria such as control, setup effort, data boundary, speed, risk; calm recommendation; accepted tradeoff.

**Likely failures:** pricing-table layout; bright winner badge; generic pros/cons cards; source product names.

## tn-incident-activity-brief

**Prompt:** Create a technical brief for an alert investigation. Show what happened, the agent’s evidence, the suspected cause, and the next action for the on-call engineer.

**Eval axis:** generation

**Claims under test:** summary/evidence/action stack, rows carry progress, terminal panels contain real material, muted semantic state.

**Strong output traits:** concise current state; log snippets; investigation timeline rows; suspected cause; one next action.

**Likely failures:** red alarm dashboard; prose-only summary; decorative log filler; cinematic noir treatment.
