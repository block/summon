## Composition

Badges carry planning state so the user reads less and understands faster — they
are a state language, not decoration.

**Badges carry state, not confetti.** Build every state chip from the signature
pill recipe — `--garden-chip-radius` pills faced with `--garden-chip-face`, inked in
`--garden-chip-ink`, bordered by `--garden-chip-edge`, and padded with
`--garden-chip-pad` — so a row of chips reads like a status sentence, not decorative
confetti. Use them to communicate time, effort, readiness, season, confidence,
energy, or waiting. Keep labels short and practical — Ready, Needs one thing, Low
effort, Can wait, Energy, Season, Timing — and reach for a chip only when the state
actually helps the user decide what to do now. Never badge-as-confetti: a note
without a meaningful state needs no chip.

**State color is reserved for real state.** Use the semantic tokens — soft green
(`--color-success`) for ready, clay (`--color-danger`) for a real blocker, muted
blue (`--color-info`) for context, warm warning (`--color-warning`) for timing —
only when the state genuinely calls for it. Readiness, waiting, blockers, and
timing are real planning state; soft green is a planning emphasis color, not a
default decoration for every element. Most chips sit quiet on `--garden-chip-face`
and let the green accent stay reserved for the chosen next step; the rare "check
first" caution rides `--garden-clay-accent`, an earthy hand-thrown terracotta
rather than a shrill alert red.

These chips ride on the [note panels](note-panels), mark readiness on the
[good-enough choice and can-wait notes](good-enough), and appear across every
surface — the readiness chips of the [planner](planner) and [routine](routine),
the staged badges of the [staged-plan](staged-plan), and the friendly recommended
badge of the [comparison](comparison) (never a severe winner label).
